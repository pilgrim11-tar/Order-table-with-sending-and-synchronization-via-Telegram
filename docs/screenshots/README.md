# Скріншоти для інструкції

У цю папку додаються скріншоти, на які посилається файл [../SETUP_UA.md](../SETUP_UA.md).

## Іменування файлів

Використовуйте саме ці назви:

- `01-orders-sheet.png`
- `02-open-apps-script.png`
- `03-code-gs.png`
- `04-script-properties.png`
- `05-columns-j-k-l.png`
- `06-trigger-onedit.png`
- `07-test-telegram.png`
- `08-workflow-check.png`
- `09-script-id.png`

## Що має бути на кожному скріншоті

### `01-orders-sheet.png`

Перший лист `Замовлення` з колонками:

- `A ID`
- `B Дата`
- `C Назва`
- `D Фірма виробник`
- `E Каталожний номер`
- `F Прев'ю (Фото)`
- `G Кількість`
- `H Терміновість`
- `I Зауваження`
- `J Надіслати`
- `K Стан замовлення`
- `L Пакет`

### `02-open-apps-script.png`

Меню Google Sheets:

- `Розширення`
- `Apps Script`

### `03-code-gs.png`

Відкритий редактор Apps Script з файлом `Code.gs`.

### `04-script-properties.png`

Сторінка `Project Settings` з блоком `Script properties`.

На скріншоті не повинно бути реального токена.

### `05-columns-j-k-l.png`

Налаштування:

- чекбокс у `J`
- випадаючий список у `K`
- службова колонка `L`

### `06-trigger-onedit.png`

Створення тригера:

- function: `onEdit`
- source: `From spreadsheet`
- type: `On edit`

### `07-test-telegram.png`

Результат запуску:

- `testManagerTelegram()`
- `testEngineerTelegram()`

### `08-workflow-check.png`

Робочий сценарій:

- кілька позначених рядків у `J`
- пакет у `L`
- оновлений лист `Документ для підпису`

### `09-script-id.png`

Сторінка `Project Settings` з полем `Script ID`.

## Рекомендації

- Формат: `PNG`
- Ширина: бажано не менше `1400px`
- Без приватних токенів, `chat_id`, пошти чи зайвих персональних даних
- Якщо є чутливі дані, закрийте їх перед збереженням скріншота
