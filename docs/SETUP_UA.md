# Покроковий запуск

Ця інструкція описує перший запуск проєкту в Google Sheets, Apps Script і синхронізацію через `clasp`.

## 1. Підготувати таблицю

У Google Sheets має бути 2 листи:

- `Замовлення`
- `Документ для підпису`

На листі `Замовлення` мають бути колонки:

- `A`: ID
- `B`: Дата
- `C`: Назва
- `D`: Фірма виробник
- `E`: Каталожний номер
- `F`: Прев'ю (Фото)
- `G`: Кількість
- `H`: Терміновість
- `I`: Зауваження
- `J`: Надіслати
- `K`: Стан замовлення
- `L`: Пакет

Скріншот:

![Лист Замовлення](screenshots/01-orders-sheet.png)

## 2. Відкрити Apps Script

У таблиці відкрийте:

- `Розширення` → `Apps Script`

Після цього відкриється прив'язаний до таблиці проєкт Apps Script.

Скріншот:

![Відкрити Apps Script](screenshots/02-open-apps-script.png)

## 3. Вставити код

1. Відкрийте файл `Code.gs`.
2. Вставте код із репозиторію.
3. Переконайтесь, що назви листів збігаються:
   - `Замовлення`
   - `Документ для підпису`
4. Збережіть проєкт.

Скріншот:

![Вставити Code.gs](screenshots/03-code-gs.png)

## 4. Налаштувати Script Properties

У Apps Script відкрийте:

- `Project Settings`
- блок `Script properties`

Додайте 3 ключі:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_MANAGER_CHAT_ID`
- `TELEGRAM_ENGINEER_CHAT_ID`

Скріншот:

![Script Properties](screenshots/04-script-properties.png)

## 5. Налаштувати лист Замовлення

### Колонка J

У колонці `J` має бути чекбокс.

Шлях:

- `Дані` → `Перевірка даних`
- тип: `Checkbox`

### Колонка K

У колонці `K` має бути випадаючий список:

- `Отримано`
- `В роботі`
- `Видано`

### Колонка L

У колонці `L` не потрібна формула. Вона заповнюється скриптом автоматично.

Скріншот:

![Налаштування J K L](screenshots/05-columns-j-k-l.png)

## 6. Створити тригер

У Apps Script відкрийте:

- `Triggers`
- `Add Trigger`

Параметри:

- Function: `onEdit`
- Deployment: `Head`
- Event source: `From spreadsheet`
- Event type: `On edit`

Скріншот:

![Тригер onEdit](screenshots/06-trigger-onedit.png)

## 7. Перевірити Telegram

Запустіть вручну:

- `testManagerTelegram()`
- `testEngineerTelegram()`

Якщо функції відпрацювали успішно, значить:

- токен валідний
- `chat_id` правильні
- `UrlFetchApp` працює

Скріншот:

![Тест Telegram](screenshots/07-test-telegram.png)

## 8. Перевірити робочий сценарій

1. Створіть кілька рядків на листі `Замовлення`.
2. Для рядків однієї дати поставте галочки в `J`.
3. Скрипт має:
   - сформувати пакет
   - відправити одне повідомлення менеджеру
   - записати ID пакета в `L`
   - зняти галочки в `J`
4. Перевірте лист `Документ для підпису`.
5. Змініть статус у `K`.
6. Інженеру має прийти повідомлення про новий статус.

Скріншот:

![Робочий сценарій](screenshots/08-workflow-check.png)

## 9. Синхронізація через clasp

### Встановлення

Потрібен Node.js, після чого встановіть `clasp`:

```bash
npm install -g @google/clasp
```

### Авторизація

```bash
clasp login
```

### Налаштування

У репозиторії вже є файл `.clasp.json`.

У ньому треба замінити:

- `PASTE_YOUR_SCRIPT_ID_HERE`

на реальний `scriptId` вашого Apps Script проєкту.

### Де взяти scriptId

У Apps Script:

- `Project Settings`
- `Script ID`

Скріншот:

![Script ID](screenshots/09-script-id.png)

### Завантаження коду в Apps Script

З папки репозиторію:

```bash
clasp push
```

### Завантаження коду з Apps Script локально

```bash
clasp pull
```

## 10. Перевірка перед бойовим запуском

- перевипустіть Telegram токен, якщо він десь публікувався
- не комітьте реальні токени в git
- перевірте назви листів без зайвих пробілів
- перевірте, що тригер саме `On edit`, а не `On change`
